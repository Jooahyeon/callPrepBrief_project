# 콜 프렙 브리프 시스템 가이드

영업 미팅 전, 4개의 Claude Code 에이전트가 협업해 **콜 프렙 브리프**(고정 포맷 한국어 사전 준비 자료)를 자동 생성하는 시스템입니다. 이 문서는 전체 워크플로우·입출력 규칙·사용법을 정리합니다.

---

## 1. 에이전트 구성

| 에이전트 | 역할 |
|----------|------|
| `call-prep-orchestrator` | 전체 워크플로우 오케스트레이션. 직접 리서치/작성은 하지 않고, 하위 3개 에이전트를 **엄격한 순서**로 위임·조율 |
| `call-prep-info-collector` | 브리프에 필요한 입력값 수집·검증 (고객사, 담당자, 미팅 목적, 제안 제품 등) |
| `client-research-collector` | 고객사 웹 리서치 (사업 개요, 최근 뉴스/이슈, 조직 규모, 공공기관 여부·규제, 주요 인물 등) — **출처 URL 포함** |
| `call-prep-brief-generator` | 검증된 입력값 + 리서치 결과를 결합해 고정 포맷 브리프 생성 및 파일 저장 |

---

## 2. 워크플로우 (순차 실행)

```
사용자 요청
   │
   ▼
[Step 1] call-prep-info-collector ── 입력값 수집·검증
   │   (입력 충분성 게이트: 필수값 누락 시 다시 수집)
   ▼
[Step 2] client-research-collector ── 고객사 웹 리서치 (생략 불가)
   │
   ▼
[Step 3] call-prep-brief-generator ── 브리프 생성 + docs/briefs/ 저장
   │
   ▼
브리프 출력 + "수정할 부분 있으면 말씀해주세요!"
```

**핵심 규칙**
- **순차 실행**: 이전 단계가 완료·검증되기 전에는 다음 단계로 넘어가지 않습니다.
- **입력 충분성 게이트**: 필수 입력값이 빠지면 브리프 생성 전에 `call-prep-info-collector`로 되돌아갑니다.
- **리서치 생략 불가**: 사용자가 많은 맥락을 줘도 웹 리서치 단계는 건너뛰지 않습니다 (최신·검증 정보로 브리프 강화).
- **수정 요청 처리**: 브리프 전달 후 수정 요청이 오면 변경 성격에 따라 해당 단계(입력/리서치/포맷)만 다시 실행합니다.

근거: `.claude/agents/call-prep-orchestrator.md` (워크플로우/규칙), `.claude/agents/call-prep-brief-generator.md` (포맷/저장).

---

## 3. 입력값 규칙

**필수**
- 고객사명
- 담당자 (이름/직책)
- 미팅 목적
- 제안 제품/서비스

**선택**
- 이전 대화 이력
- 고객 관심사항 / 우려사항

> 필수값이 하나라도 빠지면 브리프를 생성하지 않고 추가 수집을 요청합니다.

---

## 4. 출력 고정 포맷

```
[ 콜 프렙 브리프 ]
================================
고객사    :
담당자    :
미팅 목적 :
================================
[ 핵심 어젠다 ]
1.
2.
3.

[ 예상 질문 & 답변 ]
Q1.
A1.
Q2.
A2.

[ 주의사항 ]
-

[ 이번 미팅 성공 기준 ]
-

[ 출처 / 참고 링크 ]
- (제목): (URL)
================================
```

- 모든 내용은 한국어로 작성 (고유명사 제외).
- 헤더·구분선(`====`)·번호·불릿 마커를 변경하지 않습니다.
- `[ 출처 / 참고 링크 ]`에는 리서치에 사용한 소스 URL을 빠짐없이 보존합니다.
- 브리프 끝에는 항상 `수정할 부분 있으면 말씀해주세요!`를 덧붙입니다.

---

## 5. 사용법 (VSCode)

1. VSCode에 **Claude Code 확장** 설치 후 이 프로젝트 폴더(`callPrepBrief_project`)를 엽니다.
2. `.claude/agents/`의 4개 에이전트가 자동 로드됩니다.
3. `call-prep-orchestrator`를 호출하면 입력 수집 → 리서치 → 브리프 생성까지 전 과정이 진행됩니다.
4. 생성된 브리프는 화면에 출력되고, `docs/briefs/브리프_<고객사>_<YYYYMMDD>.md`로 저장됩니다.

> 빈 템플릿과 저장 규칙은 [`briefs/README.md`](./briefs/README.md), [`briefs/_TEMPLATE.md`](./briefs/_TEMPLATE.md) 참고.
